Next.js is one of the most popular tools in modern web development, and for good reason — it takes React, the world's most widely used UI library, and wraps it in a powerful framework that handles all the hard parts of building a real web application. If you've ever wondered how professional developers go from an idea to a fast, production-ready website, Next.js is a huge piece of that puzzle.

## What Is a Web Framework, and Why Do We Need One?

When you build a website with plain HTML, CSS, and JavaScript, you're responsible for everything: organizing your files, wiring up navigation between pages, fetching data from a server, optimizing images so they load quickly, and much more. That's a lot of decisions and a lot of boilerplate code before you've written a single line of your actual app.

A **web framework** is a collection of tools, conventions, and pre-built solutions that handle those common problems for you. Instead of reinventing the wheel every time, you follow the framework's patterns and focus on what makes your app unique.

Think of it like building a house. You could quarry your own stone, mill your own lumber, and forge your own nails — or you could start with a solid foundation and pre-cut materials. A framework is that foundation.

## What Is Next.js?

Next.js is a **React framework for building full-stack web applications**. It was created by [Vercel](https://vercel.com) and is now used by companies like Netflix, TikTok, Twitch, and The Washington Post.

Here's the big picture of how it works:

1. You write your UI using **React components** and **Next.js conventions**
2. Next.js handles routing, rendering, data fetching, and optimization automatically
3. Your users get a fast, polished application in their browser

You get the flexibility of React with the structure and power of a full framework — without having to configure everything from scratch.

## Your First Taste: A Next.js Page

Every page in a Next.js app is just a React component exported from a file. Here's what the simplest possible page looks like:

```jsx
// app/page.tsx
export default function HomePage() {
  return (
    <main>
      <h1>Hello, Next.js!</h1>
      <p>This is my first Next.js page.</p>
    </main>
  )
}
```

That's it. Drop that file in the `app/` folder and Next.js automatically serves it at the root URL (`/`). No routing configuration, no imports from a router library — the file system *is* the router.

<Note>
  Don't worry if this looks unfamiliar right now. By the end of this course, writing components like this will feel completely natural.
</Note>

## Which Version of Next.js Are We Using?

This course uses **Next.js 14 or later** with the **App Router** — the modern, recommended way to build Next.js applications. You may come across older tutorials that reference the "Pages Router" (where pages live in a `pages/` folder instead of `app/`). The App Router is the current standard, and that's what we'll use throughout this course.

<Warning>
  If you find a tutorial online that uses `pages/index.js` or `getServerSideProps`, it's using the older Pages Router. The concepts overlap, but the syntax is different. Stick with the App Router patterns you'll learn here.
</Warning>

## The Next.js Ecosystem at a Glance

Next.js doesn't exist in isolation. It sits at the center of a modern development stack:

| Layer | Tool |
|---|---|
| UI Library | React |
| Framework | Next.js |
| Styling | Tailwind CSS (used in this course) |
| Language | TypeScript |
| Deployment | Vercel (or any Node.js host) |

You'll work with all of these throughout the course. For now, just know that they all fit together cleanly — Next.js is designed to work well with each of them.
